После запуска сервера интерактивная документация доступна по адресам:
​Swagger UI: http://127.0.0.1:8000/api/docs/ — тестирование запросов в реальном времени.
​Schema: http://127.0.0.1:8000/api/schema/ — техническая спецификация YAML.
​
Основные Эндпоинты:
​Аутентификация (JWT)
​POST /api/token/ — получение токенов доступа (Access) и обновления (Refresh).
​POST /api/token/refresh/ — получение нового Access-токена.

​(Posts)
​GET /api/v1/posts/ — список всех опубликованных статей.
​POST /api/v1/posts/ — создание новой статьи (требуется JWT).
​GET /api/v1/posts/{id}/ — детальный просмотр статьи.
​PUT /api/v1/posts/{id}/ — редактирование (только для автора).
​DELETE /api/v1/posts/{id}/ — удаление (только для автора).

​Комментарии (Comments)
​GET /api/v1/posts/{id}/comments/ — список комментариев к конкретному посту.
​POST /api/v1/posts/{id}/comments/ — добавление комментария (требуется JWT).

Как протестировать через Swagger:
Авторизуйтесь через /api/token/, используя логин и пароль созданного суперпользователя.
Скопируйте полученный access токен.
Нажмите кнопку Authorize вверху страницы Swagger UI.
Введите в поле: Bearer <ваш_токен> (через пробел).
Теперь вам доступны защищенные методы (POST, PUT, DELETE).
